# [二維矩形背包問題 2D Rectengular Knapsack Problem]

## Proposal Report

### 動機與目標
<!-- 說明為什麼想做這個專題 -->
動機：我一直對數學邏輯問題很有興趣，總覺得解開複雜問題很有成就感。上學期在課程中聽到了經典的「背包問題 (Knapsack Problem)」，覺得它非常有意思。剛好我平常有在玩電玩遊戲，其中一款裡面有背包機制，玩家從地圖上蒐集不同大小及價值的矩形物品，並將它們裝入空間有限的背包內，每次都需要思考怎麼把它們「塞得剛剛好」且「價值最高」，要花很多時間手動調整。我就想到如果能用演算法來幫忙，是不是就能找到一種很厲害的裝法？因此決定結合這兩者，研究如何在有限的矩形空間裡，挑選並裝入總價值最高的物品組合。

專題目標：<br>
一、建立二維幾何排版系統： 實作能避免矩形物品重疊與超出邊界的排版引擎，並具備基礎的視覺化展示功能。<br>
二、開發高價值篩選策略： 針對「價值」與「空間利用率」兩項指標，設計多種演算法，找出在不同場景下能極大化總價值的擺放邏輯。<br>
三、空間管理優化與效能評估： 減少填裝死角與碎片化空間，並對比不同演算法在運算時間與收益結果上的差異，尋求穩定且高效的解決方案。<br>
四、夢幻目標(基本上不可能)：給定一個m*n矩形背包，以及若干個大小各異的矩形物品(長寬皆不大於min(m,n))，從這些矩形物品裡找出能放進背包且總價值最高的組合

### 預期功能
<!-- 列出預計實作的功能 -->
一、空間放置<br>
1.實作放置演算法，處理物件放置邏輯<br>
2.能避免邊界超出與物品重疊的問題<br>
3.(進階)物件可旋轉後再放置<br>
4.(進階)能避免碎片化空間，盡量讓剩餘空間聚在一起，甚至能在放新物品的時候重新整理空間來確認它能不能被放進去

二、價值最佳化運算<br>
1.實作多種排序規則(譬如價值密度 V/A高的優先級高)，進行初步篩選<br>
2.實作不同演算法，尋找能放進背包且總價值越高越好的組合<br>
3.比較不同演算法的結果，並從中挑選價值最高的組合

三、視覺化<br>
1.將結果用圖片輸出，並標示各物品編號和價值<br>
2.計算空間利用率、平均單格價值等等

### 與課程的關聯
<!-- 你的專題可能涉及哪些資料結構或演算法概念？為什麼？ -->
一、資料結構<br>
1.類別與物件(Class / Objects)：<br>
封裝矩形物品的屬性，如寬度、高度、價值、座標及旋轉狀態，用於管理不同物件的狀態。

2.動態陣列 (Dynamic Arrays / Vectors)<br>
儲存「待放入物品清單」與「已放置物品紀錄」， 方便進行排序。

3.空閒矩形清單 (Free Rectangles List)<br>
用來儲存背包中「尚未被佔用的矩形區域」，每次放新東西時，程式可以去這個清單找「放得下的空格」，而不是亂塞。

二、幾何與排版演算法<br>
1.碰撞偵測 (Collision Detection)：<br>
利用座標邊界判定檢查兩個矩形是否重疊，確保物品不會重疊。

2.啟發式放置演算法 (Placement Heuristics)：<br>
如 Bottom-Left (BL) 或 MaxRects，決定物品進入背包後先放在哪裡。例如 BL 演算法會讓物品盡量往左下角靠攏，以留出連續的大空間。

三、優化與搜尋演算法<br>
1.貪婪演算法 (Greedy Algorithm)：<br>
根據特定權重（如價值密度 V/A）進行降冪排序，依序嘗試置入。提供一個快速且不差的初步解。

2.進階（如遺傳演算法 Genetic Algorithm, 選位）：<br>
用更進階的方法嘗試尋找最佳的「物品放入順序」。挑戰比貪婪演算法更高的總價值，展現演算法的優化能力。
