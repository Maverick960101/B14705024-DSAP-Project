# [二維矩形背包問題 2D Rectengular Knapsack Problem]

## Proposal Report

### 動機與目標
<!-- 說明為什麼想做這個專題 -->
動機：我一直對數學邏輯問題很有興趣，總覺得解開複雜問題很有成就感。上學期在課程中聽到了經典的「背包問題 (Knapsack Problem)」，覺得它的邏輯非常有意思。剛好我平常有在玩電玩遊戲，其中一款裡面有背包機制，玩家從地圖上蒐集不同大小及價值的矩形物品，並將它們裝入空間有限的背包內，每次都需要思考怎麼把它們「塞得剛剛好」且「價值最高」，要花很多時間手動調整。我就想到如果能用演算法來幫忙，是不是就能找到一種很厲害的裝法？因此決定結合這兩者，研究如何在有限的矩形空間裡，挑選並裝入總價值最高的物品組合。

夢幻目標：給定一個m*n矩形背包，以及若干個大小各異的矩形物品(長寬皆不大於min(m,n))，從這些矩形物品裡挑選出能放進背包且總價值最高的組合<br>
不過上面的目標應該是過於困難，所以我的最終目標主要是寫出一些不錯的演算法，極大化背包內的價值
### 預期功能
<!-- 列出預計實作的功能 -->
1.空間放置<br>
實作放置演算法，處理物件放置邏輯<br>
能避免邊界超出與物品重疊的問題<br>
(進階)物件可旋轉後再放置<br>
(進階)能避免碎片化空間，盡量讓剩餘空間聚在一起，甚至能在放新物品的時候重新整理空間來確認它能不能被放進去

2.價值最佳化運算<br>
實作多種排序規則(譬如金額面積比值高的優先級高)，進行初步篩選<br>
實作不同演算法，尋找能放進背包且總價值越高越好的組合<br>
比較不同演算法的結果，並從中挑選價值最高的組合

3.視覺化<br>
將結果用圖片輸出，並標示各物品編號和價值<br>
計算空間利用率、平均單格價值等等

### 使用技術
<!-- 使用的語言、框架、工具等 -->
語言部份主要以python實作，假如有處理到python執行效率較低的部分例如遞迴，可能那個部分會以C++實作<br>
工具預計使用Visual Studio Code和GitHub，可能會用到Matplotlib、NumPy、Rectpack等函式庫
### 時程規劃
<!-- 各週預計完成的進度 -->
week8-9 不考慮價值，嘗試實作二維背包問題(2D Knapsack Problem)的一個子問題，即給定一個m*n矩形背包，以及若干個大小各異的矩形物品(長寬皆不大於min(m,n))，從這些矩形物品裡挑選出能放進背包，且讓空間利用率達到最高<br>
week10-12 根據上面得到的結果，並開始考慮各物品價值，實作一些基礎的演算法(譬如貪婪演算法)，使背包內的價值不算太差<br>
week 13-15 嘗試實作一些進階的演算法，並與之前的基礎演算法比較，期許背包內價值能顯著提升
