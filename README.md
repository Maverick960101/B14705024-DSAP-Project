# 二維矩形裝箱問題 2D Rectangle Bin Packing Problem

## Proposal Report

### 動機與目標
<!-- 說明為什麼想做這個專題 -->
動機：我一直對數學邏輯問題很有興趣，總覺得解開複雜問題很有成就感。上學期在課程中聽到了經典的「背包問題 (Knapsack Problem)」，覺得它非常有意思。剛好我平常有在玩電玩遊戲，其中一款裡面有背包機制，玩家從地圖上蒐集不同大小及價值的矩形物品，並將它們裝入空間有限的背包內，每次都需要思考怎麼把它們「塞得剛剛好」且「價值最高」，要花很多時間手動調整。我就想到如果能用演算法來幫忙，是不是就能找到一種很厲害的裝法？因此決定結合這兩者，研究如何在有限的矩形空間裡，挑選並裝入總價值最高的物品組合。

專題目標：<br>
一、建立二維幾何排版系統： 實作能避免矩形物品重疊與超出邊界的排版引擎，並具備基礎的視覺化展示功能。<br>
二、開發高價值篩選策略： 針對「價值」與「空間利用率」兩項指標，設計多種演算法，找出在不同場景下能極大化總價值的擺放邏輯。<br>
三、空間管理優化與效能評估： 減少填裝死角與碎片化空間，並對比不同演算法在運算時間與收益結果上的差異，尋求穩定且高效的解決方案。<br>
四、夢幻目標(基本上不可能)：給定一個m*n矩形背包，以及若干個大小各異的矩形物品(長寬皆不大於min(m,n))，從這些矩形物品裡找出能放進背包且總價值最高的組合<br>
五、先以實作已存在的演算法為主，訓練自己的程式開發能力，待能寫出幾種主流演算法後，再來實踐自己的改良想法是否有效

### 競品比較
目前實作：First Fit Decreasing Height (FFDH)，邏輯為遍歷所有層，放入第一個塞得下的位置。<br>
比較項目：Next Fit Decreasing Height (NFDH)，只看目前這一層，塞不下就開新層，不再回頭。<br>
         Best Fit Decreasing Height (BFDH)，遍歷所有層，放入塞入後剩餘空間最小的位置。<br>
         斷頭台演算法(Guillotine Algorithm)，每次放入物品後切割剩餘空間，讓剩餘空間變成多個小矩形來維護。<br>
目前的演算法相比另外的幾個，空間使用效率並不算高，但在「開發速度與難度」與「效能回報」之間取得了最佳平衡點，因此選擇此作為第一個實作的項目。

## Final Report

### 專案說明
<!-- 完整描述你的專案做了什麼 -->
本專案旨在研究二維裝箱問題階段，並未如一開始所想導入價值，此問題目標為給定一個m*n矩形背包，以及若干個大小各異的矩形物品(長寬皆不大於min(m,n))，判斷是否能全部放入背包內，我實作了兩種在研究此問題時常見的演算法FFDH和斷頭台演算法，物品的排序統一按照物品高度降序排序，從左下角優先放起。並界定數據範圍，範圍內隨機生成10000筆數據來檢驗成功率。<br>
FFDH演算法：多樓層架子演算法。最大物件放左下角，其高度定為「一樓天花板」，後續物件往右緊貼橫排，當右邊塞不下時，從一樓重新往上找空位；若全滿，則在頂樓天花板上方開闢新樓層。在均質資料下表現穩定，但在面對極端大小混雜的資料集時，容易產生無法回填的真空帶。<br>
斷頭台演算法：分為水平切割和動態切割，每次放置物件時，會將因放入物品而變成非矩形空間的空間切成兩個矩形。<br>
水平切割：每次都將剩餘空間水平切割，打破架子限制的切分演算法。但在隨機高度降序測資下，容易將空間切碎為高窄的廢料矩形。<br>
動態切割：本專案的核心成果。在每次放置物件時，即時計算剩餘空間的長寬比，動態決定垂直或水平切分，確保剩餘空間最為方正，能容納大部分一般物件，避免長寬比極端空間。<br>

我用了不同的資料來測試演算法效能，當放入多個大小較相近的物品時，FFDH和斷頭台的動態切割版本都獲得了約90%的成功率，但斷頭台的水平切割版本只有約46%的成功率；當物品分布為少數幾個大物品和多個小物品時，FFDH成功率約為54%，斷頭台的水平切割版本約為59%，斷頭台的動態切割版本則高達75%。

本專案完全基於 Python 內建函式庫開發，僅需安裝繪圖工具 matplotlib，程式會隨機生成固定範圍內的數據來比較演算法差異，也有使用者可自行輸入數據來觀察差異的版本。

### 與課程的關聯總結
<!-- 總結你的專題與進階程式設計及資料結構課程之間的關聯 -->
資料結構：<br>
二維幾何的資料表達：本專案不依賴第三方幾何套件，完全使用原生Python的List與Tuple來動態儲存與更新空間狀態。<br>
多樓層狀態維護：在FFDH中，利用巢狀列表實作動態查找清單，克服了單一變數無法記憶歷史樓層的缺陷。<br>
空間邊界管理：在斷頭台演算法中，將剩餘空間維護成一個動態的矩形清單，隨著物件置入，對List進行精準的pop與append操作，體會到記憶體空間動態配置的邏輯。<br>
演算法：
啟發式演算法與貪婪策略：2D裝箱屬於NP-hard問題，本專案實作了First-Fit(時間複雜度O(n^2))與Best-Area-Fit兩種經典貪婪策略，體會到用局部最佳解去逼近全域近似解的過程。<br>
分治與空間切分：斷頭台演算法的核心在於將一個大矩形問題，拆解為兩個不重疊的小矩形子問題。在Debug的過程中，深入理解了邊界條件只要重疊，就會引發演算法崩潰的致命性。<br>
隨機模擬：利用 random 函式庫實作大數據壓力測試，透過10000次隨機實驗，量化分析「資料排序特徵（高度降序）」與「空間切割機制（動態二分）」之間的關係，將演算法優化建立在客觀的數據實證上。<br>
